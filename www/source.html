Building Regina from Source

Building Regina from its sources can be either trivially simple or quite painstaking depending upon your system configuration. With each new release the scripts that automatically detect your system configuration become more sophisticated, and hopefully Regina is thus becoming simple to build on more and more systems.

If Regina does not build directly out of the box for you, please drop me an email (including your config.log if possible) so that the automatic detection scripts can be made to work with your system and others like yours for the next release.

Basically the steps are as follows.

  • Make sure you have installed all the necessary tools and libraries as listed in the system requirements table below.
  • Download the source archive regina-version_src.tgz from the Regina download site.
  • Unpack this source archive; it will expand into a directory regina-version/ on your machine.
  • From within this directory run ./configure. This will run the automatic detection scripts that search for the necessary tools and libraries on your system and work out how they can be used to build Regina.
  • If the configure succeeds, run make to build Regina. Then, as root, run make install to install Regina on your system.
That's all there is to it. A full set of instructions is included in the file README.txt which is bundled with the archive. Please do read it; it's short and it contains some important information.

If you are having trouble compiling or running Regina, you may check the troubleshooting page to see if your problem is discussed there. Of course you are also welcome to mail either myself or the users' mailing list (regina-user@lists.sourceforge.net) for assistance.

System Requirements

The tools and libraries that you will need installed before you can build Regina are listed in the table below. All of these tools and libraries are available for most GNU/Linux distributions and are easily ported to other similar systems.

It is highly recommended that you download prebuilt packages of these tools for your GNU/Linux distribution, since this is most likely to provide standard installations in standard locations that Regina can easily find and use. Where you will find such packages depends upon your distribution. For instance, Debian packages can all be found on the usual Debian servers, Fedora Core packages can be found on the Fedora download servers and so on.

The following table lists each requirement along with its Debian package name and a commonly-used RPM package name (though these tend to vary between different RPM-based distributions). It also lists the website from which each piece of software can be obtained directly if no pre-built packages are available.

Please note that if you get an error at build time saying that it cannot link with -lfoo for some library foo, you probably just need to install another development package on your system. For instance, Fedora users who get the error "/usr/bin/ld: cannot find -lselinux" need to install the additional package libselinux-devel, and users who get the error "/usr/bin/ld: cannot find -ljpeg" need to install the package libjpeg-devel.
