What is Regina?

Regina is essentially a topological calculator with a bent towards normal surface theory. It allows 3-manifolds to be created, manipulated and their properties of interest determined.

Its primary uses (in my opinion) include:

  • the ability to examine 3-manifolds too complex to study by hand;
  • the ability to use the census feature to run tests on all 3-manifolds of a given type;
  • the ability to bring my PhD thesis closer to completion.

Download

You can download prebuilt binary distributions for GNU/Linux or Windows systems, or you can download the entire source tree to compile Regina for yourself. All downloads are available from the SourceForge project site. Be aware of the GNU General Public License, which is also contained in the downloadable archives.

Before you download, be sure to visit the system requirements page. Regina needs several other pieces of software installed before it will run.

  • Binaries:
    • Download the binary archive regina-version_bin_machine.tgz corresponding to your machine type.
      • GNU/Linux users will want regina-version_bin_i386-linux-gnu.tgz. Note that these binaries are built against libc6, libstdc++2.10 and libgmp3.
      • Windows users will want regina-version_bin_i686-cygwin.tgz. Note that if you plan to use the batch file regina-version/regina.bat, you will need to edit it before you first run it. The batch file itself contains instructions for how to do this.
    • Unpack the archive. The application will be extracted into directory regina-version/. Instructions for running the application are included in regina-version/README.txt.
  • Sources:
    • Download the source archive regina-version_src.tgz.
    • Unpack the archive. The sources will be extracted into directory regina-version/. Instructions for compiling the application are included in regina-version/README.txt.

To extract archive stuff.tgz, type tar -zxf stuff.tgz.

Documentation

The Regina Reference Manual can be viewed here online.

There are several ways of obtaining the reference manual on your machine. The first of these methods is recommended. All downloads are available from the SourceForge project site.

  • You can use your favourite web browser to view the documentation.
    • Download the archive regina-version_doc.tgz or regina-version_doc.zip.
    • Unpack this archive, which will extract the documentation into directory regina-version/docs/.
    • If you wish to use your favourite web browser to view help from within Regina (which is recommended, since the internal JavaHelp system has problems with some of the HTML), run Regina and use the Options-Help Browser menu item to specify the command used to start your web browser and the location where the documentation is unpacked (such as /path/to/regina-version/docs).
  • You can use Regina's embedded JavaHelp browsing system.
    • Download the jar archive regina-version_doc-internal.jar.
    • The embedded help browser can use this jar as is; there is no need to unpack it. Simply rename it to regina-docs.jar and place it in the same directory as the other jar files (regina.jar, regina-jni.jar and so on; this directory is probably regina-version/bin/).
  • You can generate the documentation from the sources.
    • Check the system requirements page to ensure that you have installed all the tools you will need.
    • Move to the main regina-version/ directory (where you have unpacked the sources) and type make bindocs.
    • The documentation will be generated in the subdirectory docs/. The title page for the reference manual will be docs/index.html. A jar that the embedded JavaHelp browser can use will be created as bin/regina-docs.jar. Unix man pages will be generated in the subdirectory docs/ (regina.1, regina.conf.5 and so on).

Contact / Comments / Suggestions

If you have any suggestions, problems, bugs, wishes, frustrations or otherwise miscellaneous comments, I would really love to hear them. This program is permanently under development and I would like to hear what people want out of it.

Alternatively, if you have written additions yourself and think they could be useful in the general release, I would love to hear about them. The more the merrier! It's probably worth mailing me first though in case I or someone else have already written said addition and it's currently being debugged and two days away from release.

Incidentally, even if you have no comments to make, I'd like to hear from anyone using the program - even if it's just to say hi - mainly because I'm interested to see who's using it and how it's being used.

The best way to contact me is by email at benb@acm.org.
