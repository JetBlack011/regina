Installing DocBook Tools

Visit the system requirements page and download all of the software you need; this should include the DocBook SGML DTD, the DocBook DSSSL stylesheets, the ISO entity sets and Jade.

Make a directory on your system for SGML-related tools (such as c:\sgml and unpack each of these software bundles in its own directory beneath it. For instance, you might have c:\sgml\docbook, c:\sgml\docbook-dsssl, c:\sgml\isoents and c:\sgml\jade.

Set the following two environment variables. Windows NT and 2000 users should do this through the control panel (select System and look for Environment Variables); Windows 95 and 98 users should do this in autoexec.bat (which will require a reboot, go figure).

  • SGML_CATALOG_FILES - This should be a semicolon-separated list of the catalog files installed with the DocBook SGML DTD, the DocBook DSSSL stylesheets and Jade. You should prepend .\catalog to this list. An example is:
    .\catalog;c:\sgml\docbook\docbook.cat;c:\sgml\docbook-dsssl\catalog;c:\sgml\jade\catalog
  • SGML_SEARCH_PATH - This should be a semicolon-separated list of all the directories in which you have just unpacked software. You should prepend . to this list. An example is:
    .;c:\sgml\docbook;c:\sgml\docbook-dsssl;c:\sgml\isoents;c:\sgml\jade

But we're not done yet. The ISO entity files will all need to be renamed. They currently have names like ISOamsa, ISOgrk1 and the like. You need to change these names to iso-amsa.gml, iso-grk1.gml and the like. You can either do this by hand (there were 19 files last time I looked), or if you have Cygwin installed with the appropriate packages (see the Cygwin support page for details) then you can fire up a bash shell, change to the corresponding directory (remember that C:\ can be found in bash as /cygdrive/c/) and enter the following command:

  • for i in ISO*; do mv $i `echo $i | sed -e s/ISO/iso-/`.gml; done
And we're done! Now you can run up a Cygwin shell, go to the main Regina source directory, run make docshtml and see what happens!
