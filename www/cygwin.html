Installing Cygwin

Cygwin provides a windows port of many of the development tools that one generally finds on a GNU/Linux system. You can download it from http://www.cygwin.com/.

You should then be able to build Regina from within a cygwin shell the same way you would build Regina under linux.

When installing Cygwin you are offered a choice of several packages to install. To build Regina you will absolutely need packages bash, binutils, cygwin, fileutils, findutils, gcc, gzip, make, sed, sh-utils, tar, textutils and w32api (although many of the other packages are also useful). Please mail me if you discover any additional packages that need to be added to this list.

GNU Multiple Precision Arithmetic Library

To build the Regina calculation engine you will need the GNU Multiple Precision Arithmetic Library (libgmp). There are two methods of obtaining this library:
  • Sources: You can download the libgmp sources from ftp://ftp.gnu.org/gnu/gmp/ and build the library yourself. Unpack the sources, move into the main source directory (the directory containing configure) and then issue the commands:
    • ./configure
    • make
    • make install
  • Binaries: You can download prebuilt libgmp binaries. Visit the Regina download site and look in the support/cygwin module. You should find a prebuilt libgmp cygwin binary archive (gmp-version_cygwin-bin.tgz). Unpack this archive in your root cygwin directory using the following commands (issued from within a cygwin shell):
    • cd /
    • tar -zxf path/to/gmp-version_cygwin-bin.tgz
    where path/to is changed to reflect where you saved the archive and version is changed to reflect the name of the archive. You should now have the library /usr/local/lib/libgmp.a and the header /usr/local/include/gmp.h.
