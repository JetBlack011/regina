Program Options

The order of precedence for program options set using multiple methods is as follows:

  1. Options specified as command-line or applet parameters;
  2. Options set using the RegConf configuration utility;
  3. Default option values as described below.

User Interface (Standalone Application)

Regina can be run from the command line with the following arguments:

regina [--gui|--console|--text] [--jni|--corba] [--jnilibname=name] [-ORBInitialHost host] [-ORBInitialPort port]

Interface options:

  • --gui
    Run with the full GUI. This is the default.
  • --console
    Run as a graphical JPython console, without loading the full GUI.
  • --text
    Run as a text JPython console, with no GUI whatsoever.

Engine access options:

  • --jni
    Access the C++ calculation engine as a shared library using JNI. This is the default.
  • --corba
    Access the C++ calculation engine as a CORBA server application, possibly running on a different machine.

JNI engine options:

  • --jnilibname=name (only for JNI engine access)
    Use name as the name of the JNI calculation engine library to load. name is the platform-independent library name, so the library will in fact be called libname.so or name.dll or so on. The default name is regina-engine-jni.

CORBA engine options:

  • -ORBInitialHost host (only for CORBA engine access)
    Specifies that the CORBA name service is running on the machine host. The default host is localhost.
  • -ORBInitialPort port (only for CORBA engine access)
    Specifies that the CORBA name service is listening on port port on the given host. The default port is 8088.

User Interface (Web Page Applet)

The parameters available when running Regina as an applet on a web page are as follows. Note that the C++ calculation engine will always be accessed as a CORBA server application.

  • Docs
    The value of Docs is the URL of the directory in which the unpacked documentation can be found by the web browser. This is the directory beneath which normal/docs/index.html can be found. The URL may be relative to the location of the applet's web page. The default value of this parameter is docs.
  • Engine
    This parameter specifies how the C++ calculation engine will be accessed. A value of corba specifies that the engine will be accessed as a CORBA server application, possibly running on a different machine. Currently this is the only supported value.
  • Interface
    This parameter specifies which style of interface to use when running Regina. A value of gui specifies the full Regina GUI. Currently this is the only supported value.
  • ORBInitialHost
    The value of ORBInitialHost is the machine on which the CORBA name service is running. The default host is localhost.

    If included, parameter org.omg.CORBA.ORBInitialHost will override this value.

  • ORBInitialPort
    The value of ORBInitialPort is the port on which the CORBA name service is listening on the given host. The default port is 8088.

    If included, parameter org.omg.CORBA.ORBInitialPort will override this value.

Calculation Engine CORBA Server

The calculation engine CORBA server can be run from the command line with the following arguments:

regina-engine-corba [-ORBInitialHost host] [-ORBInitialPort port]

  • -ORBInitialHost host
    Specifies that the CORBA name service is running on the machine host. The default host is localhost.
  • -ORBInitialPort port
    Specifies that the CORBA name service is listening on port port on the given host. The default port is 8088.
